About the models

Owners are principals: identifiers representing users and other authority entities such as groups or roles.

At any moment, a process has the authority to act on behalf of some(possibily empty) set of principals.

A label L contains a set of owners, owners(L); these are the principals whose data was observed in order to obtain that value.

For each owners O, the label contains a set of readers, readers(L,O); these are the principals that O allows to observe the value;

Observations happen when values are written to output channels. Each output channels C has an associated set of readers; these are the principals who will be able to observe information written to that channel. A value can be written to a channel only if each reader of the channel has the authority to act for some reader in the effective readers set of the value's label.

Declassification can be performed only by a process with the authority to act for the owner whose policy is being changed.

Limitation of the model

  • Adding readers. We should be able to add a reader r for some owner o if O already allows some reader r' that r acts for
  • Replacing owners. We should be able to replace an owner o' with some principal o that can act for o'
